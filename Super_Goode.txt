Super Goode Map - Full Project Overview

Project Goal
Build a static web map application that shows restaurants reviewed by Phil Goode, including map pins, review scores, subtitles/categories, directions links, review video links, nearby sorting, and mobile-friendly UI.

1. Core Architecture
The project is a fully static web app with no backend and no database. It runs from static files on GitHub Pages and uses JSON plus GitHub automation for updates.

Main flow:
User Interface
-> JSON dataset
-> GitHub automation
-> Google Sheet / CSV intake

2. The Map Application
The public website is a map-based restaurant browser.

It loads restaurant data from:
data/locations.json

For compatibility it also falls back to:
locations.json

The UI includes:
- restaurant pins
- clickable markers
- zoom and navigation
- fit mapped pins
- restaurant cards
- name, score, subtitle, address, distance, directions, and review video actions

3. Mobile Experience Improvements
The UI was tuned for phones and small screens.

Improvements included:
- responsive layout
- larger touch targets
- map/list behavior improvements
- mobile map focus toggle
- location-aware sorting
- distance display
- Phil branding and color treatment

4. Data Extraction and Cleanup
The original map data was extracted into a shared JSON dataset and cleaned.

Stored fields include:
- name
- score
- subtitle
- reviewUrl
- directionsUrl
- address
- city
- state
- lat
- lng
- sourceType
- confidence
- notes

Cleanup work included:
- correcting mismatched coordinates
- correcting bad addresses
- removing duplicate rows
- removing test rows like TEST BURGER and Test Pizza Place
- preserving stronger existing data when merging updates

5. Admin Intake Tool
The repo includes a single-file admin helper:
admin/add-review.html

It supports:
- restaurant name
- score
- subtitle
- review URL
- directions URL
- address
- city
- state
- notes

It can output:
- pretty JSON
- one-line JSON snippet

6. Data Merge System
The merge script:
scripts/update_locations.js

Purpose:
Merge intake rows into the master dataset safely.

Features:
- prevents duplicates
- normalizes names
- preserves stronger existing data
- applies manual fixes last
- blocks obvious test/demo rows
- prints Added / Updated / Skipped / Ambiguous / Errors

7. Google Sheet / CSV Integration
The repository uses a Google Sheet as the intake source, published as CSV.

The sync script:
scripts/sync_sheet.js

It:
- downloads the published CSV from GOOGLE_SHEET_CSV_URL
- parses the CSV
- keeps only approved rows
- blocks obvious test entries
- feeds the approved rows into the merge script

8. GitHub Automation
GitHub Actions workflow:
.github/workflows/sync-sheet.yml

It:
- runs on a schedule and manually
- downloads the published CSV
- merges approved rows
- updates data/locations.json
- commits the changes back to the repo

12. Current System Behavior
Adding restaurants:
Admin fills out the form or adds rows in the sheet.

Approval:
Only rows marked approved are published.

Sync:
GitHub Actions downloads the CSV, merges rows, and commits the updated data.

Site update:
GitHub Pages serves the latest static JSON dataset.

14. Notes
This repo intentionally avoids a backend, database, or dynamic server dependency.
The public app reads the updated JSON file only.
The CSV export and text overview are local repo artifacts for backup and documentation.
