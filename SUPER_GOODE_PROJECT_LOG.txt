Super Goode Map Project Log

Purpose
This repo hosts a static GitHub Pages map for Phil Goode's restaurant reviews.
The site reads a shared static dataset and shows restaurant pins, scores, subtitles,
directions links, review video links, nearby sorting, and mobile-friendly controls.

What was built
1. Static dataset-backed map app
   - The public site loads restaurant data from data/locations.json.
   - The app falls back to the root locations.json for compatibility.
   - Map pins, popups, list cards, score chips, and mobile controls are all in index.html.

2. Data extraction and cleanup
   - Extracted the original map data into a shared JSON dataset.
   - Preserved score, subtitle, directionsUrl, reviewUrl, coordinates, confidence, and notes.
   - Corrected mislocated entries when the address and coordinates did not match.
   - Removed test entries like TEST BURGER and Test Pizza Place from the published dataset.

3. Mobile and branding improvements
   - Added Phil Goode branding and icon styling.
   - Tuned the color palette to match the source site.
   - Improved mobile layout, map/list behavior, and action button sizing.
   - Added and refined location-aware features:
     - Use my location
     - Near me
     - distance display
     - fit mapped pins

4. Admin intake workflow
   - admin/add-review.html
     - Single-file, no-build admin helper.
     - Generates valid JSON for a new restaurant.
     - Supports name, score, subtitle, reviewUrl, directionsUrl, address, city, state, and notes.
     - Can generate either pretty JSON or a one-line snippet.

5. Merge and sync scripts
   - scripts/update_locations.js
     - Merges approved intake rows into data/locations.json.
     - Preserves stronger existing data.
     - Avoids duplicates through name normalization.
     - Applies manual-fixes.json last.
     - Prints Added / Updated / Skipped / Ambiguous / Errors.
     - Blocks obvious temp/test rows.
   - scripts/sync_sheet.js
     - Reads a published CSV URL from GOOGLE_SHEET_CSV_URL.
     - Parses the CSV.
     - Keeps only approved rows.
     - Reuses the existing merge script.
     - Blocks obvious temp/test rows.

6. GitHub Actions automation
   - .github/workflows/sync-sheet.yml
     - Runs on a schedule and manually.
     - Downloads the published CSV.
     - Merges approved rows into data/locations.json.
     - Commits updated data back to the repo.
   - This keeps the site static and GitHub Pages compatible.

  - data/locations.json is the main shared dataset used by the site.
  - data/manual-fixes.json stores permanent overrides.
  - data/new-reviews.json is the intake file for manual additions.
  - root locations.json remains as a compatibility mirror.

GitHub work done

  - Fixed the shared-data loader so GitHub Pages actually reads data/locations.json.
  - Added the mobile and branding improvements and pushed them live.
  - Added the admin intake helper and merge script.
  - Added the GitHub Action workflow for CSV-based sheet sync.
  - Switched sync ingestion from the Google Sheets API to a published CSV URL secret.
  - Removed temporary/test restaurant rows from the published dataset.
  - Removed the footer note text from the site.
  - Kept the site fully static with no backend or database dependency.

Important current behavior
  - The site is static and works on GitHub Pages.
  - New approved sheet rows can be synced into the dataset automatically.
  - Obvious test/demo rows are blocked from merge and sync.
  - Manual fixes remain the last override layer.
  - Real restaurants are preserved unless there is an explicit targeted cleanup.

Current status
